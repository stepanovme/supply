# Жизненный цикл заявки

## 1. Создание

Пользователь создаёт заявку, заполняет шапку и позиции.

## 2. Проверка и отправка

После заполнения заявка отправляется на согласование назначенным пользователям.

## 3. Согласование

Согласующие подтверждают или отклоняют заявку. При большинстве голосов статус меняется автоматически.

## 4. Работа со счетами

По заявке прикрепляются счета, выполняется распознавание и сопоставление позиций.

## 5. Исполнение

После прохождения этапов заявка переходит в работу и далее в завершённое состояние.

## Статусы

| Этап | Что означает |
|---|---|
| Новая | Черновик, можно редактировать |
| На согласовании | Ожидает решений согласующих |
| В работе | Согласована, передана в исполнение |
| Отклонена | Не прошла согласование |
| Завершена | Полностью исполнена |
